## 1. 사업 개요

### 1-1. 서비스 소개

TubeWatch는 유튜브 채널 데이터를 AI로 분석하여 크리에이터에게 **실행 가능한 성장 인사이트**를 제공하는 SaaS 플랫폼이다.

기존 유튜브 스튜디오나 분석 툴이 '수치 조회'에 그치는 것과 달리, TubeWatch는 단일 분석 실행으로 4개의 구조화된 진단 결과를 도출하고, 오늘 당장 실행할 수 있는 액션 플랜까지 자동 생성한다.

### 1-2. 핵심 가치 제안

| 기존 서비스 | TubeWatch |
|------------|-----------|
| 조회수·구독자 수치 나열 | 성과 패턴 추출 → 성공 공식 도출 |
| 플랫폼 평균 비교 | 내 채널 내부 데이터 기반 진단 |
| 분석 결과 제공 후 종료 | 우선순위별 실행 액션 리스트 제공 |
| 전체 채널 재분석 | 모듈 단위 부분 업데이트 |

## 2. 핵심 기술 개요

### 2-1. 기술 구조 요약

```
[데이터 수집 레이어]
  YouTube Data API v3 → 채널 메타데이터 + 영상 50개 자동 수집

[분석 처리 레이어]
  단일 분석 실행 엔진 (analysis_run_id 기준)
  → Partial Update (모듈 단위 독립 재실행)
  → Engine Version 관리 (버전 기반 캐시 무효화)

[데이터 정규화 레이어]
  Raw API 응답 → Normalized Dataset
  → 페이지별 ViewModel 파생 (단일 기준 데이터 유지)

[AI 분석 레이어]
  LLM (대규모 언어 모델) 기반 진단 생성
  → 8대 분석 신호 구조화 출력
  → 채널 DNA (성공 패턴) 자동 추출

[저장 및 트랜잭션 레이어]
  Supabase PostgreSQL (Snapshot + Module Output)
  → Atomic Credit 트랜잭션 (reserve / confirm / rollback)
  → Run Tracking (progress_step 기반 실시간 상태 추적)

[서비스 레이어]
  Next.js 풀스택 / Vercel Edge 배포
  → Ghost Data + Hybrid Cache (UX 연속성 보장)
```

**기술적 핵심 포인트:**

#### ① 단일 분석 실행 기반 다중 뷰 파생 아키텍처
- `analysis_run_id` 또는 `snapshot_id`를 단일 기준점으로 고정
- 모든 분석 페이지(채널진단 / Channel DNA / 액션플랜 / 트렌드)가 동일한 Run 데이터에서 ViewModel을 파생
- 페이지 간 데이터 정합성 100% 보장 (채널 간 데이터 혼재 원천 차단)

#### ② Partial Update 모듈 시스템
- 분석을 전체 재실행 없이 모듈(action_plan, channel_dna, next_trend, seo_lab) 단위로 독립 재실행
- LLM API 비용 40~60% 절감 효과
- `analysis_runs.analysis_type` 필드로 실행 유형 추적

#### ③ Engine Version 기반 캐시 무효화
- `analysis_runs.engine_version` / `module_outputs.module_version` 필드 체계
- 엔진 버전 변경 시 시간 기반 캐시(12시간)와 무관하게 즉시 업데이트 권장
- 프롬프트 품질 개선 즉시 사용자 반영 가능

#### ④ Atomic Credit 트랜잭션 시스템
- `reserve()` → 분석 시작 전 크레딧 선점
- `confirm()` → 분석 완료 후 확정 차감
- `rollback()` → 분석 실패 시 자동 복구
- `run_id ↔ credit_reservation_id` 1:1 연결로 정산 오류 추적

#### ⑤ Channel DNA 자동 추출 알고리즘
- 채널 내 상위 성과 영상들의 공통 패턴 자동 분류
- 8대 분석 신호 체계: 콘텐츠 주제 일관성 / 시청자 반응 구조 / 채널 활동 패턴 / SEO 최적화 상태 / 성장 모멘텀 / 구독 전환 구조 / 업로드 주기 / 키워드 밀도
- 레이더 프로필 시각화 (channelActivity / audienceResponse / contentStructure 3축)

## 3. 시장 분석

### 3-1. 시장 규모

| 구분 | 내용 |
|------|------|
| 국내 유튜브 채널 수 | 100만 개 이상 (추정) |
| 월 1회 이상 업로드 활성 채널 | 약 20~30만 개 |
| 국내 MCN 등록 채널 | 약 3~5만 개 |
| 글로벌 유튜브 채널 수 | 5억 개 이상 |

### 3-2. 목표 시장 및 세분화

**1차 시장 (B2C):** 구독자 1,000명~100만 명 규모의 성장 추구 크리에이터
- 채널 운영 데이터를 활용한 개선 의지가 있는 층
- 유튜브 스튜디오 데이터만으로는 인사이트 도출 어려움을 체감하는 층

**2차 시장 (B2B):** MCN 기획사, 브랜드 채널 운영 기업, 디지털 마케팅 에이전시
- 다수 채널 일괄 진단 수요
- 채널 성과 리포트 정기 수령 수요

### 3-3. 경쟁사 분석

| 구분 | TubeWatch | 경쟁사 A (해외 일반 분석 툴) | 유튜브 스튜디오 |
|------|-----------|---------------------------|----------------|
| 데이터 출처 | 내 채널 내부 패턴 | 외부 비교 지표 중심 | 플랫폼 원시 지표 |
| AI 진단 | ✅ (LLM 기반) | 제한적 | 없음 |
| 실행 액션 생성 | ✅ (P1~P3 우선순위) | 없음 | 없음 |
| 채널 DNA 추출 | ✅ (독자 알고리즘) | 없음 | 없음 |
| 원화 결제 | ✅ | ❌ | N/A |
| 국내 맞춤 서비스 | ✅ | ❌ | ❌ |

### 3-4. 경쟁 우위

1. **내부 패턴 기반 진단**: 다른 채널과 비교가 아닌, 내 채널 과거 데이터에서 성공 공식을 추출 → 채널 규모 무관 유효
2. **실행 가능한 결과**: 분석에서 끝나지 않고 "오늘 바로 할 일" P1~P3 액션 리스트 자동 생성
3. **단일 분석 다중 활용**: 1회 분석 실행으로 4개 전문 리포트 파생 → API 비용 최적화
4. **국내 특화**: 원화 결제, 한국어 진단 문장 생성, 국내 유튜브 생태계 맥락 반영
