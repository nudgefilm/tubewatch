## 2. 프로그램 기능 설명

### 2-1. 서비스 목적

TubeWatch는 유튜브 채널 URL 입력만으로 채널의 과거 영상 데이터를 자동 수집·분석하여, 인공지능이 생성한 다차원 진단 리포트와 맞춤형 콘텐츠 기획안을 제공하는 SaaS(Software as a Service) 프로그램이다.

### 2-2. 핵심 기능 4가지

#### ① Channel Analysis — 채널 건강검진 리포트
- 조회수·구독자·참여율을 종합 점수(0~100)로 산출
- 업로드 패턴, 썸네일·제목 클릭률 영향도 분석
- 최근 성과 추이 시각화 (트렌드 차트)

#### ② Channel DNA — 채널 정체성 자동 추출
- 채널 고유 성공 패턴(DNA) 도출 (30개 신호 / 9개 지표 / 7개 패턴 기반)
- 레이더 프로필 시각화 (채널 활동성 / 시청자 반응 / 콘텐츠 구조 3축)
- 타겟 시청자 페르소나 자동 분류

#### ③ Action Plan — 실행 계획 자동 생성
- 분석 결과를 P1(긴급) / P2(중요) / P3(권장) 우선순위별 실행 액션으로 변환
- 유튜브 스튜디오에서 즉시 수정 가능한 결손 항목 제시
- 업로드 전 체크리스트 자동 생성

#### ④ Next Trend — 다음 영상 방향 제안
- 채널 내부 성과 패턴 기반 차기 영상 주제 후보 추천 (최대 5개)
- 시청 지속 시간 패턴(Retention Curve) 분석을 통한 챕터별 스크립트 구조 생성
- 추천 이유 + 예상 효과 + 신호 강도 등급 함께 제공

### 2-3. 기술적 구성 요소

| 구성 요소 | 설명 |
|-----------|------|
| 데이터 수집 모듈 | YouTube Data API v3 기반 채널/영상 메타데이터 수집 |
| 데이터 정규화 모듈 | 시계열 기울기 변화율 계산, 비정형 텍스트 벡터 임베딩 |
| AI 분석 엔진 | LLM(대규모 언어 모델) 기반 하이브리드 추론 알고리즘 |
| Partial Update 시스템 | 모듈(action_plan / channel_dna / next_trend) 단위 독립 재실행 |
| Atomic Credit 트랜잭션 | reserve / confirm / rollback 3단계 원자적 크레딧 처리 |
| 단일 분석 기준 아키텍처 | analysis_run_id 기반 모든 페이지 ViewModel 일관성 보장 |
| Hybrid Cache 시스템 | Ghost Data + 신규 분석 결과 seamless 교체 UX |
| 진단 라벨 생성기 | makeDiagnosticLabel() — 분석 문장을 8~22자 진단 라벨로 변환 |

### 3-2. 핵심 알고리즘 설명 (제출용 요약)

**① 채널 DNA 추출 알고리즘**

채널의 최근 영상 50개 데이터에서 성과 상위 그룹과 하위 그룹을 분리하고, 상위 그룹에서 공통적으로 반복되는 특징 패턴을 30개 신호 체계(콘텐츠 주제 일관성 / 시청자 반응 구조 / 채널 활동 패턴 / SEO 최적화 상태 / 성장 모멘텀 / 구독 전환 구조 / 업로드 주기 / 키워드 밀도 등)에 기반하여 자동 분류한다. 이를 채널 고유 "DNA"로 정의하고, 3축 레이더 프로필로 시각화한다.

**② 진단 라벨 생성 알고리즘 (makeDiagnosticLabel)**

LLM이 생성한 자연어 진단 문장을 UI 카드 제목(8~22자)으로 변환하는 규칙 기반 알고리즘이다. 접두 제거 → 전체 패턴 매칭 → 주제어+상태어 조합 → 폴백의 4단계로 처리한다.

**③ 단일 기준 분석 파생 구조**

모든 분석 페이지는 `analysis_run_id` 또는 `snapshot_id` 단일 기준점에서 ViewModel을 파생하며, 페이지마다 독립적으로 데이터를 fetch하는 구조를 원천 차단하여 정합성을 보장한다.

**④ Atomic Credit 트랜잭션**

분석 실행 전 크레딧을 선점(reserve)하고, 완료 후 확정(confirm), 실패 시 복구(rollback)하는 3단계 원자적 처리로 이중 차감 및 환불 오류를 방지한다.
